The Full Story
Problem → Solution → Real Business Impact
What was broken, exactly what the automation does, and what changed after deployment.

What Was Wasting Hours Every Day

The business runs inbound service calls (HVAC, Pest Control, Plumbing) across the USA. Every call was recorded — but extracting insights from those recordings required someone to manually listen, judge, and summarize each one. With 50–100+ calls daily, that was completely unscalable.

  • 5+ Hours of Audio Sat Unanalyzed Daily

    No one had time to listen to every call recording. Quality issues, missed bookings, and agent problems were only discovered days later — if at all.

  • No Structured MIS from Call Data

    Management had no way to see call outcomes at a glance — which calls were converted, which agents failed to assist, which customers were in wrong areas. All of this required manual logging.

  • Agent Performance Was Invisible

    Without per-call verdicts and issue attribution (Buyer Side vs Seller Side), there was no objective way to evaluate call quality or identify which agents needed coaching.

  • MIS Reports Were Always Delayed

    Any reporting on call performance required manually writing up summaries — a process that took hours and was always a day or more behind real-time operations.

What the Automation Does — Step by Step

I built a 14-node N8N workflow that runs on a schedule, picks up call recording URLs from Google Sheets, processes them through AI, and writes structured MIS results back — zero human involvement required.

  • Step 1 — Auto-fetch Recording URLs from Google Sheets

    The workflow triggers on a schedule, reads all pending call rows from Google Sheets, filters only valid recordings, and loops through them one by one in batches.

  • Step 2 — Transcribe Audio via OpenAI Whisper

    Each recording URL is fetched as audio and sent to the Whisper API for transcription — converting spoken calls into clean, accurate text in seconds.

  • Step 3 — GPT Call Auditor Agent Analyzes the Transcript

    A custom GPT agent (acting as a professional call quality analyst) reads the transcript and produces: a 10-word summary, a verdict from 9 predefined categories, and an issue attribution (Buyer Side / Seller Side / No Clear Issue).

  • Step 4 — Structured Output Written Back to Google Sheets

    The parsed JSON output is written directly back to the corresponding row in Google Sheets — giving management an instantly readable MIS report without touching a single file.

  "verdict"  : "Accepted & Converted"                                             // One of 9 categories
  "issue_side": "No Clear Issue"                                             // Buyer Side / Seller Side / No Clear Issue
}
Verdict Categories
9 Call Outcome Classifications
The GPT agent assigns every call to one of these verdict categories based on the transcript.
✓ Accepted & Converted

Call successful — service booked or confirmed by customer.

ℹ Customer Inquiry Only

Informational call — customer asked questions without booking.

⚠ Looking for Another Provider

Customer not interested — comparing with competitors.

⚠ Can't Serve in Area

Service not available in the customer's location.

✕ Seller Failed to Assist

Agent unable to resolve the issue or mishandled the call.

✕ Caller Hang Up

Customer disconnected early before resolution.

✕ Target Transfer Not Available

Transfer failed — no receiving agent was available.

ℹ Wrong Number Dialed

Customer reached the wrong number accidentally.

ℹ No Interaction Both Sides

Call connected but no conversation took place.

Technical Details
What's Inside the Pipeline
A breakdown of every node and its role in the automation.

Schedule Trigger

Runs the entire pipeline automatically on a defined schedule — no manual execution needed.

Google Sheets Integration

Reads pending call rows, writes structured results back, and logs errors to a separate row — fully automated data management.

Batch Loop with Error Handling

Processes recordings one by one in a loop. If a row fails (bad URL, API error), it logs the error and continues to the next — the pipeline never breaks.

Recording Filter Node

Skips rows with missing or already-processed recordings — only valid, unanalyzed calls are sent through the AI pipeline.

Whisper API Transcription

Each audio URL is fetched and sent to OpenAI Whisper for accurate speech-to-text conversion — handles any call length.

GPT Call Auditor Agent

A custom-prompted LangChain agent acting as a professional call quality analyst — produces summary, verdict, and issue side for every transcript.

Structured Output Parser

Forces GPT output into a strict JSON schema — ensuring summary, verdict, and issue_side are always cleanly parseable for Sheets write-back.

5-Second Rate Limit Wait

A built-in wait node between each call prevents OpenAI API rate limit errors — keeping the pipeline stable across large batches.
